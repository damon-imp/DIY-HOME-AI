    How It Works
    Cameras capture snapshots every 15-60 minutes. Each image is sent to Claude's vision model with a room-specific prompt. The AI returns structured tasks: "Put the two plates from the counter into the dishwasher." Tasks are pushed to your phone and logged for weekly pattern reports.

    02 — Hardware Setup

    Cameras & Server

    Camera Placement Guide

    Kitchen: Mount under a cabinet or on top of the refrigerator, angled to capture counter surfaces, the stove, and the sink. The AI needs to see where dishes pile up, food gets left out, and whether the stove appears active.

    Living Room: Place on a bookshelf, entertainment center, or high corner. Angle to capture the main seating area, coffee table, and floor space. Goal: detect clutter, items on the floor, cups/plates left out.

    Office / Entryway: Desk-level for office, shelf-height for entry. Goal: mail pileup, packages, shoes/coats, desk clutter, cables.

    Pro Tip
    Take a photo from each camera angle with your phone before mounting permanently. If you cannot clearly see the problem areas in the phone photo, the camera will not capture them either.

    Initial Camera Setup via Wyze App

    Download the Wyze app (iOS/Android) and create an account.

    Plug in your first camera. Wait 20 seconds until the status light flashes red.

    Press and hold the SETUP button on the bottom for 3 seconds until you hear a voice prompt.

    Follow in-app instructions to connect the camera to your 2.4GHz WiFi.

    Name each camera (Kitchen, Living Room, Office) for easy identification.

    Repeat for all three cameras. Verify live feeds work in the app.

    Critical
    Write down each camera's IP address. Open your router admin panel (usually 192.168.1.1), find connected devices, and note IPs. Set static DHCP reservations so they don't change on reboot. You will need these IPs for software configuration.

    Flash RTSP Firmware

    RTSP firmware lets you pull video/snapshots directly from cameras over your local network, bypassing Wyze cloud.

    Go to support.wyze.com and download the RTSP firmware for Wyze Cam v3.

    Format a 32GB microSD card as FAT32.

    Copy the firmware file (demo_wcv3.bin) to the root of the SD card. Not in a folder.

    Power off the camera (unplug it). Insert the microSD card.

    While holding the SETUP button, plug the camera back in. Hold for 3-6 seconds.

    LED turns solid blue, then flashes. Wait 3-4 minutes for flash to complete. Camera reboots.

    In Wyze app → Camera Settings → Advanced → Enable RTSP.

    Note the RTSP URL displayed (format: rtsp://CAMERA_IP/live). Repeat for all cameras.

    Server Setup

    The Beelink arrives with Windows 11. You have two paths:

    Option A: Install Ubuntu (Recommended)

    Download Ubuntu Desktop 24.04 LTS from ubuntu.com/download.

    Flash it to a USB drive using Rufus (Windows) or Balena Etcher (Mac).

    Plug USB into Beelink. Power on, press F7 repeatedly for boot menu.

    Select USB drive. Choose "Erase disk and install Ubuntu."

    Set username, password, hostname (e.g., "hup-server").

    Connect to WiFi or ethernet. Open Terminal:

    sudo apt update && sudo apt upgrade -y

    Option B: Keep Windows

    The scripts work on Windows too. Install Python 3.11+ from python.org and ffmpeg from ffmpeg.org (add to PATH). Setup script won't work, so install dependencies manually.

    Verify Camera Connectivity

    Confirm your server can reach each camera on the network before installing software.

    ping 192.168.1.101    # Kitchen camera IP
ping 192.168.1.102    # Living room camera IP
ping 192.168.1.103    # Office camera IP

    If pings succeed, test a snapshot grab:

    sudo apt install ffmpeg

ffmpeg -y -rtsp_transport tcp -i rtsp://192.168.1.101/live \
  -frames:v 1 -q:v 2 test_kitchen.jpg

    If this produces a test_kitchen.jpg, open it and verify the angle. Repeat for each camera.

    Checkpoint
    If all three cameras produce valid JPEG files, your hardware layer is complete. You are ready for software.
    03 — Software Installation

    Deploy the Code

    Get Your API Keys

    Anthropic API Key

    Go to console.anthropic.com. Create an account or sign in.

    Go to Settings → API Keys → "Create Key." Name it "HUP AI."

    Copy the key (starts with sk-ant-...). Save it securely. Cannot view it again.

    Settings → Billing → Add payment method. Load $20 to start.

    Pushover Keys

    Download Pushover app on your phone ($5 one-time).

    Go to pushover.net, create account. Copy your User Key from the dashboard.

    Click "Create Application/API Token." Name it "HUP AI." Copy the API Token.

    Install & Configure

    Transfer the project archive to your server and run setup:

    mkdir -p ~/hup_project
tar -xzf diy-hup-ai.tar.gz -C ~/hup_project/
cd ~/hup_project
chmod +x setup.sh
./setup.sh

    The setup script installs Python, ffmpeg, creates a virtual environment, installs packages, creates data directories, and sets up a systemd service for auto-start.

    Configure Settings

    nano ~/hup_project/config.py

    Update these critical fields:

    SettingWhat to Change
    ANTHROPIC_API_KEYYour sk-ant-... key
    CAMERAS → urlEach camera's actual IP address
    PUSHOVER_USER_KEYYour Pushover user key
    PUSHOVER_API_TOKENYour Pushover app token
    SNAPSHOT_INTERVAL_MINUTES15, 30, or 60 (default: 15)
    ACTIVE_HOURS_START / ENDWhen to scan (default: 7AM-10PM)
    Security
    For production use, set API keys as environment variables instead of hardcoding. The config supports this: set ANTHROPIC_API_KEY in your environment and remove the fallback string from config.py.

    05 — Go Live

    Run It 24/7

    Start the Daemon

    python hup.py daemon

    The daemon scans continuously at your configured interval, handles active hours, sends morning/evening digests, and cleans up old snapshots automatically.

    Auto-Start on Boot

    Make it survive reboots and power outages:

    Edit the systemd service with your API keys:

    sudo nano /etc/systemd/system/hup-ai.service

# Find the Environment lines and set:
Environment=ANTHROPIC_API_KEY=sk-ant-api03-your-actual-key
Environment=PUSHOVER_USER_KEY=your-pushover-user-key
Environment=PUSHOVER_API_TOKEN=your-pushover-app-token

    Enable and start:

    sudo systemctl daemon-reload
sudo systemctl enable hup-ai      # Auto-start on boot
sudo systemctl start hup-ai       # Start now
sudo systemctl status hup-ai      # Verify running

    Management Commands

    CommandWhat It Does
    python hup.py statusToday's scans, tasks, and weekly patterns
    python hup.py digestTrigger a daily summary notification
    python hup.py weeklyWeekly pattern report
    python hup.py scanForce a one-off scan right now
    sudo systemctl stop hup-aiStop the background daemon
    sudo systemctl restart hup-aiRestart after config changes
    journalctl -u hup-ai -fWatch live logs
    06 — Troubleshooting

    Fix Common Issues

    Camera Issues

    ProblemCauseFix
    FAILED in testWrong IP or RTSP offVerify IP in router admin. Re-check RTSP toggle in Wyze app.
    TIMEOUTNetwork issue or cam offlinePower cycle camera. Try -timeout 30 in ffmpeg.
    Black imageNight mode or bad angleCheck night vision settings. Ensure ambient light.
    WiFi drops2.4GHz interferenceMove router closer. Add WiFi extender. Set static channel.

    API Issues

    ProblemCauseFix
    401 UnauthorizedBad API keyRegenerate at console.anthropic.com. Update config.py.
    429 Rate LimitedToo many requestsIncrease scan interval in config.py.
    JSON parse errorMalformed AI responseRare. Check logs. Usually resolves next cycle.
    High costsInterval too frequentSwitch to 30min or 60min interval.

    Notification Issues

    ProblemCauseFix
    No notificationsWrong keys or app not installedRun test-notify. Verify keys. Reinstall app.
    DelayedPhone in DND or battery saverWhitelist Pushover in phone settings.
    Too manyEvery scan sends full reportSet NOTIFY_IMMEDIATELY_ON_HIGH = False
    07 — Future Upgrades

    Level Up

    The base system is intentionally simple. After living with it for a week or two, here are the high-value upgrades ranked by impact.

    High Impact

    Motion-Triggered Scanning

    Only scan when cameras detect motion. Cuts API costs 60-80% while increasing responsiveness. Most cameras support motion webhooks.
    High Impact

    Task Completion Tracking

    Reply "done" to notifications to mark tasks complete. Track completion rates over time. Turns passive scanner into accountability tool.
    Medium Impact

    Before / After Comparison

    After a task is marked done, capture a new snapshot and have AI verify it was completed.
    Medium Impact

    Home Assistant Integration

    Tie AI analysis into automations: lights, smart plugs, room-specific alerts via HA on the same Beelink.
    Nice to Have

    Voice Announcements

    Bluetooth speaker near each camera. TTS reads tasks aloud when high-priority items detected.
    Nice to Have

    Multi-Person Assignment

    Assign rooms/categories to different household members. Route notifications per person.
